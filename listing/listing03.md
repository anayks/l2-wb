Что выведет программа? Объяснить вывод программы. Объяснить внутреннее устройство интерфейсов и их отличие от пустых интерфейсов.

```go
package main

import (
	"fmt"
	"os"
)

func Foo() err error {
	var err *os.PathError = nil
	return err
}

func main() {
	err := Foo()
	fmt.Println(err)
	fmt.Println(err == nil)
}
```

Ответ:
```
Интерфейс - это то поведение, которому должна следовать структура.
Структура должна реализовывать те методы, под интерфейс которого она должна подходить.
Интерфейс может просчитываться в рантайме и кешироваться, в отличие от других языков

Интерфейс - это тоже структура, в котором есть два поля:
1. itable - структура, отвечающая за список методов, которые должны быть реализованы
2. data - сам указатель на структуру, интерфейс которой поддерживается

Пустой интерфейс - это поведение без каких-либо требований, поэтому под него подходит
абсолютно любой тип данных.
Интерфейс имеет только указатель на структуру без методов, в нём нет ничего сложного.

В данном примере, идет сравнение по типам - по интерфейсу (error) и по структуре (os.PathError)
В первом fmt.Println мы можем увидеть, что ошибка равна nil, а во втором неравна

Всё потому, что в данном случае err, равный nil, здесь типизирован.
Чтобы сравнить его с реальным nil, нужно преравнять его к типу указателя на эту структуру

Тип err в данной ситуации подходит под указатель на *os.PathError, а у nil конкретный тип - nil
Если сравнивать два этих типа, то они неравны
Если мы преравняем nil к типу указателя *os.PathError, то они станут равны.
```

Исправленный вариант
```go
package main

import (
	"fmt"
	"os"
)

func Foo() error {
	var err *os.PathError = nil
	return err
}

func main() {
    err := Foo()
    fmt.Println(err == (*os.PathError)(nil))
}
```
